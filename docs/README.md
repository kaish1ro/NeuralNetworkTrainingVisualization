# Neural Network Visualizer

Учебный десктопный инструмент для обучения нейросети с визуализацией в реальном времени.
Написан на C++20 и Qt6 с нуля — без TensorFlow, PyTorch и других фреймворков.

---

## Как собрать

**Зависимости:**
- Qt 6.2+ (Widgets, Core)
- Eigen 3 (header-only, уже в `external/`)
- CMake 3.19+
- MSVC 2022 или GCC 11+ / Clang 14+

**Сборка:**

```
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build
```

Или открой `CMakeLists.txt` напрямую в Qt Creator / CLion / Visual Studio.

**Датасеты** уже лежат в `resources/datasets/` — ничего скачивать не нужно.

---

## Быстрый старт

1. Запусти приложение
2. В левой панели выбери датасет — **MNIST** для рисования цифр, **Iris** для таблицы фич
3. При желании добавь/убери скрытые слои кнопкой **+ Add Layer**
4. Нажми **▶ Start Training**
5. После окончания — нарисуй цифру или введи значения фич и нажми **Predict**
6. Модель можно сохранить через **Save Model** и загрузить позже через **Load Model**

### Как Core устроен

**DenseLayer** хранит матрицу весов `W` (outputSize × inputSize), вектор смещений `b`, и активационную функцию. При `Forward` считает `z = W·x + b`, применяет активацию → `a`. При `Backward` получает градиент сверху, возвращает градиент вниз и накапливает `dW`, `db`.

**SequentialModel** — просто список слоёв. `Predict` — один проход вперёд. `Train` — прямой проход, потом `Loss::Backward`, потом `Backward` по всем слоям в обратном порядке, потом `UpdateWeights`.

**Инициализация весов:**
- **He** — для ReLU: `W ~ N(0, √(2/n_in))`. Компенсирует то, что ReLU убивает половину нейронов.
- **Xavier** — для Sigmoid/Tanh/Softmax: `W ~ N(0, √(1/n_in))`. Сохраняет дисперсию сигнала.

### Как UI взаимодействует с Core

```
ControlPanel → MainWindow → TrainingWorker (QThread)
                                   ↓ сигналы Qt
              epochFinished  → MetricsPanel (графики)
              batchProgress  → MetricsPanel (прогресс-бар)
              architectureReady → NeuralNetworkView (схема)
              analysisReady  → MetricsPanel (матрица, примеры)
              trainingFinished → MainWindow (активирует Predict)
```

`TrainingWorker` работает в отдельном потоке. UI никогда не блокируется. Сигналы передаются через `Qt::QueuedConnection` — Qt сам маршрутизирует их в главный поток.

## Руководство пользователя

### Выбор датасета

| Датасет | Вход | Выход | Описание |
|---------|------|-------|----------|
| MNIST | 784 пикселя (28×28) | 10 классов (0–9) | Рукописные цифры |
| Iris | 4 числа (длина/ширина лепестков) | 3 класса | Классика ML |
| Custom CSV | зависит от файла | зависит от файла | Свой датасет |

Для **Custom CSV**: последний столбец — метка класса (текст или число). Остальные столбцы — числовые признаки. CSVLoader сам нормализует их в `[0, 1]` и кодирует метки в one-hot.

### Архитектура сети

По умолчанию два скрытых слоя по 128 нейронов. Можно добавлять/убирать слои кнопкой **+ Add Layer** и менять:
- **Размер** — количество нейронов (1–1024)
- **Активацию** — ReLU / Sigmoid / Tanh

Входной и выходной слой определяются автоматически по датасету.

### Параметры обучения

| Параметр | Что делает | Совет |
|----------|------------|-------|
| Learning rate | Шаг градиентного спуска | 0.01 — хорошее начало. При прыжках loss — уменьши |
| Epochs | Сколько раз пройти по всем данным | 5–10 для MNIST достаточно |
| Batch size | Сколько примеров за один шаг | 32–64. Меньше — шумнее, быстрее сходится. Больше — стабильнее |
| Loss | Функция ошибки | CrossEntropy для классификации, MSE редко нужен |

### Графики

На вкладке **Metrics** два графика — loss (потери) и accuracy (точность) по эпохам. Каждый новый запуск обучения рисует новую линию поверх старых — удобно сравнивать эксперименты.

- **Loss падает** — модель обучается
- **Loss скачет** — learning rate слишком большой
- **Accuracy не растёт** — слишком маленькая сеть или неподходящая активация

Двойной клик на график — открывает увеличенный вид с зумом (Ctrl+прокрутка) и панорамированием.

### Визуализация сети

Центральная панель показывает архитектуру сети. После каждой эпохи:
- **Цвет связей** — вес (светлее = положительный, темнее = отрицательный)
- **Яркость нейрона** — его активация для последнего примера

Можно зумировать (Ctrl+прокрутка) и перетаскивать.

### Сохранение и загрузка модели

**Save Model** — сохраняет архитектуру, веса и параметры обучения в файл `.nnvm`.

**Load Model** — загружает модель. Если датасет в файле не совпадает с текущим — предложит переключиться.

При попытке начать обучение поверх существующей модели появится диалог с вопросом — можно сохранить перед перезаписью.

### Предсказание

**MNIST** — появляется холст, рисуй мышью (ЛКМ), сотри правой кнопкой. Предсказание обновляется в реальном времени.

**Iris / CSV** — появятся поля ввода для каждого признака с ползунком в диапазоне обучающих данных. Нажми **Predict**.

## Технические заметки

### Почему ReLU лучше Sigmoid

**Sigmoid** выдаёт значения в (0, 1). Производная: `σ(x)·(1-σ(x))`. При `|x| > 3` производная почти ноль — градиент не проходит через слой. Если слоёв несколько, градиент к первым слоям становится ничтожно малым. Это называется **vanishing gradient**.

**ReLU** (`max(0, x)`) имеет производную 1 при `x > 0` — градиент проходит без затухания. Нейроны с `x < 0` «умирают» (выдают 0), но на практике достаточно активных нейронов остаётся.

На графиках это видно: с ReLU loss падает быстрее и стабильнее. Особенно заметно на MNIST с тремя и более скрытыми слоями.

### Что такое backprop

Сеть делает предсказание (прямой проход). Считаем ошибку. Теперь нужно понять: как изменить каждый вес, чтобы ошибка уменьшилась?

Используем **цепное правило** дифференцирования: ошибка зависит от выхода последнего слоя, выход последнего слоя зависит от весов этого слоя и от выхода предпоследнего, и так до первого слоя. Считаем частные производные от конца к началу — это и есть backpropagation.

Каждый `DenseLayer::Backward` получает `dL/da` (градиент по своим выходам) и возвращает `dL/dx` (градиент по своим входам = выходам предыдущего слоя). Заодно накапливает `dW = dL/da · x^T` и `db = dL/da`.

### Почему MNIST — хороший датасет для старта

- **Фиксированный размер** — всегда 28×28 = 784 числа на вход
- **Балансированные классы** — примерно поровну примеров для каждой из 10 цифр
- **Достаточно сложный** — случайное угадывание даёт 10%, хорошая сеть — 97%+
- **Быстро обучается** — 5 эпох на простой сети дают уже 95%+
- **Визуально понятный** — можно нарисовать цифру и сразу проверить

Именно поэтому MNIST 40 лет остаётся стандартным тестом для новых архитектур.
